# 去掉無意義的提問句 (/zh-tw/docs/when-questions/remove-questional-sentence)



避免用無意義的話結束提問，例如`有人能幫我嗎？`或者`這有答案嗎？`。

## 為什麼這類句子沒有價值 [#為什麼這類句子沒有價值]

首先：如果你對問題的描述不是很好，這樣問更是畫蛇添足。

其次：由於這樣問是畫蛇添足，黑客們會很厭煩你 —— 而且通常會用邏輯上正確，但毫無意義的回答來表示他們的蔑視， 例如：`沒錯，有人能幫你`或者`不，沒答案`。

這類句子沒有提供任何診斷信息，也沒有說明你希望別人做什麼。它們只是在要求別人先承諾會不會幫你。

## 換成具體請求 [#換成具體請求]

與其寫 `有人能幫我嗎？`，不如寫：

* `我應該優先檢查哪一層？`
* `這個錯誤是否說明依賴版本不兼容？`
* `這個最小示例是否足以說明問題？`
* `有沒有我遺漏的常見配置項？`

這些問題能讓回答者直接進入技術判斷。

## 避免只會得到“是/否”的問題 [#避免只會得到是否的問題]

一般來說，避免用 `是或否`、`對或錯`、`有或沒有`類型的問句，除非你想得到[是或否類型的回答](https://strcat.de/questions-with-yes-or-no-answers.html)。

如果你真正需要的是解釋、方向或診斷建議，就不要把問題寫成只能回答“是”或“不是”的形式。
