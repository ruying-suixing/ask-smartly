# 禮多人不怪 (/zh-tw/docs/when-questions/courtesy-costs-nothing)



彬彬有禮，多用`請`和`謝謝您的關注`，或`謝謝你的關照`。讓大家都知道你對他們花時間免費提供幫助心存感激。

## 禮貌不能替代內容 [#禮貌不能替代內容]

坦白說，這一點並沒有比使用清晰、正確、精準且合乎語法和避免使用專用格式重要（也不能取而代之）。黑客們一般寧可讀有點唐突但技術上鮮明的 Bug 報告，而不是那種有禮但含糊的報告。（如果這點讓你不解，記住我們是按問題能教給我們什麼來評價問題的價值的）

也就是說，禮貌是加分項，不是遮羞布。一個沒有環境、沒有錯誤信息、沒有嘗試記錄的問題，哪怕寫滿“麻煩您了”，也仍然是一個難以回答的問題。

## 禮貌會影響長期回應 [#禮貌會影響長期回應]

然而，如果你有一串的問題待解決，客氣一點肯定會增加你得到有用回應的機會。

技術社區是有記憶的。別人會記得你是否尊重時間、是否反饋結果、是否在得到幫助後補充總結。客氣、誠實、及時感謝，會讓人更願意在下一次繼續理你。

## 關於“先謝了” [#關於先謝了]

（我們注意到，自從本指南發佈後，從資深黑客那裡得到的唯一嚴重缺陷反饋，就是對預先道謝這一條。一些黑客覺得`先謝了`意味著事後就不用再感謝任何人的暗示。我們的建議是要麼先說`先謝了`，**然後**事後再對回覆者表示感謝，或者換種方式表達感激，譬如用`謝謝你的關注`或`謝謝你的關照`。）

最穩妥的做法是：提問時表達對他人關注的感謝，問題解決後再明確感謝具體幫助你的人，並補充最終結果。
