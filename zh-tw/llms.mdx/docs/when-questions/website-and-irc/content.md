# 網站和 IRC 論壇 (/zh-tw/docs/when-questions/website-and-irc)



本地的用戶群組（user group），或者你所用的 Linux 發行版本也許正在宣傳他們的網頁論壇或 IRC 頻道，並提供新手幫助（在一些非英語國家，新手論壇很可能還是郵件列表），這些都是開始提問的好地方，特別是當你覺得遇到的也許只是相對簡單或者很普通的問題時。有廣告贊助的 IRC 頻道是公開歡迎提問的地方，通常可以即時得到回應。

## 新手問題先找用戶支持渠道 [#新手問題先找用戶支持渠道]

用戶群組、發行版論壇、網頁論壇和 IRC 頻道，往往比開發者郵件列表更適合入門問題。那裡的人更熟悉常見安裝、配置、環境差異和使用誤區，也更願意處理“我剛開始用”的問題。

## 發行版問題先問發行版社區 [#發行版問題先問發行版社區]

事實上，如果程序出的問題只發生在特定 Linux 發行版提供的版本（這很常見），最好先去該發行版的論壇或郵件列表中提問，再到程序本身的論壇或郵件列表提問。（否則）該項目的黑客可能僅僅回覆“使用**我們的**版本”。

發行版常常會修改編譯選項、補丁、默認配置和依賴版本。上游項目不一定知道這些差異，因此先問發行版社區通常更有效。

## 發問前搜索論壇內部 [#發問前搜索論壇內部]

在任何論壇發文以前，先確認一下有沒有搜索功能。如果有，就試著搜索一下問題的幾個關鍵詞，也許這會有幫助。如果在此之前你已做過通用的網頁搜索（你也該這樣做），還是再搜索一下論壇，搜索引擎有可能沒來得及索引此論壇的全部內容。

論壇內部搜索能發現搜索引擎還沒收錄、或被登錄權限擋住的討論。先搜索，也能讓你瞭解這個社區喜歡怎樣描述問題。

## 選擇同步或異步渠道 [#選擇同步或異步渠道]

通過論壇或 IRC 頻道來提供用戶支持服務有增長的趨勢，電子郵件則大多為項目開發者間的交流而保留。所以最好先在論壇或 IRC 中尋求與該項目相關的協助。

在使用 IRC 的時候，首先最好不要發佈很長的問題描述，有些人稱之為頻道洪水。最好通過一句話的問題描述來開始聊天。

IRC 更適合快速澄清和短互動；論壇更適合長問題、日誌、代碼和後續總結。選擇渠道時，先想清楚你的問題需要即時對話，還是需要可搜索、可沉澱的長文本。
