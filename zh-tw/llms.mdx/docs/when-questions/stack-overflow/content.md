# Stack Overflow (/zh-tw/docs/when-questions/stack-overflow)



搜索，*然後*在 Stack Exchange 問。

近年來，Stack Exchange 社區已經成為回答技術及其他問題的主要渠道，尤其是那些開放源碼的項目。

## 先搜索已有答案 [#先搜索已有答案]

因為 Google 索引是即時的，在看 Stack Exchange 之前先在 Google 搜索。有很高的幾率某人已經問了一個類似的問題，而且 Stack Exchange 網站們往往會是搜索結果中最前面幾個。如果你在 Google 上沒有找到任何答案，你再到特定相關主題的網站去找。用標籤（Tag）搜索能讓你更縮小你的搜索結果。

搜索時可以同時使用錯誤信息、庫名、語言名、版本號和關鍵 API 名稱。Stack Exchange 的歷史問題很多，重複問題通常不會受到歡迎。

## 提問時遵守平臺習慣 [#提問時遵守平臺習慣]

如果你還是找不到任何對你的問題有用的內容，請把你的問題發在與它最相關的網站上。提問的時候請善用格式化工具，尤其注意為代碼添加格式，並且添加相關的標籤（特別是編程語言、操作系統或庫/包的名稱）。當有人要求你提供更多相關信息時，請編輯你的貼子來補充它們\[譯註：而不是發一個回帖或回答！]。如果你覺得一個答案對你有幫助，點擊向上的箭頭來為它投票；如果一個答案提供了問題的正確解決方案，點擊投票按鈕下方的對勾來將它標記為正解。

這不是形式主義。格式、標籤、編輯補充、投票和接受答案，都是 Stack Exchange 用來維護知識庫質量的機制。

## 選擇正確的網站 [#選擇正確的網站]

Stack Exchange 已經成長到[超過一百個網站](https://stackexchange.com/sites)，以下是最常用的幾個站：

* Super User 是問一些通用的電腦問題，如果你的問題跟代碼或是寫程序無關，只是一些網絡連線之類的，請到這裡。
* Stack Overflow 是問寫程序有關的問題。
* Server Fault 是問服務器和網管相關的問題。

如果你把問題發錯站點，即使問題本身不錯，也可能被關閉或遷移。先確認站點範圍，比發完以後再解釋要有效得多。
