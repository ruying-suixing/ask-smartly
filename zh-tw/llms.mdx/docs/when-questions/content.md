# 正在提問時 (/zh-tw/docs/when-questions)



真正開始提問時，你要做的不只是把問題發出去。你需要選擇合適的場合、寫清楚標題、使用可讀格式、提供事實、描述目標，並讓回答者能快速判斷自己是否幫得上忙。

## 提問時的核心原則 [#提問時的核心原則]

當你提出問題的時候，請先表明你已經做過準備。這能幫助別人判斷你不是一個不勞而獲、隨手浪費他人時間的提問者。如果你還能說明自己在準備過程中**學到了什麼**，效果會更好；我們更樂於回答那些表現出能從答案中繼續學習的人。

一個好的問題通常具備這些特徵：

* 發在正確的社區或討論區。
* 標題能概括目標、環境和異常。
* 正文格式清楚，方便引用和回覆。
* 描述事實，而不是隻給猜測。
* 說明目標，而不是隻描述你卡住的步驟。
* 展示你已經搜索、閱讀、嘗試和排除過什麼。

## 如何閱讀本章 [#如何閱讀本章]

本章的每一節都處理提問時的一個具體動作：選論壇、寫標題、描述問題、貼代碼、避免無效句式、保持禮貌等。你不必一次記住所有規則，但在真正發問前，至少應該逐項檢查一遍。

提問質量不是靠客氣話堆出來的，而是靠準確的信息、明確的邊界和可協作的態度建立起來的。
