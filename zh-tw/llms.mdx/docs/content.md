# 提問的智慧 (/zh-tw/docs)



這份文檔討論的不是“怎樣顯得禮貌”，而是怎樣把一個混亂的問題，整理成值得他人投入判斷的問題。

在技術社區裡，提問是一種協作邀請。你給出的事實越清楚，推理越誠實，邊界越分明，回答者就越容易與你站到同一張地圖前，判斷下一步該往哪裡走。

## 從哪裡開始 [#從哪裡開始]

如果你第一次閱讀，可以先從“簡介”和“在提問之前”開始，理解技術社區為什麼如此看重準備工作。真正準備發問時，再進入“正在提問時”，逐項檢查你的標題、上下文、症狀、目標和嘗試。

如果你已經得到了回覆，也不要急著結束。後續的總結、感謝、歸檔和反饋，同樣會影響別人是否願意在下一次繼續幫助你。
