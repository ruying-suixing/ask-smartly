# 如果還是搞不懂 (/zh-tw/docs/how-to-read-answer/if-still-not-understand)



如果你看不懂回應，別立刻要求對方解釋。像你以前試著自己解決問題時那樣（利用手冊，FAQ，網絡，身邊的高手），先試著去搞懂他的回應。如果你真的需要對方解釋，記得表現出你已經從中學到了點什麼。

## 先消化，再追問 [#先消化再追問]

回答者給你的可能只是一個關鍵詞、一個方向或一個判斷。你需要先圍繞這個線索做基本功：查手冊、讀 FAQ、搜索舊討論、試驗命令或詢問身邊更熟悉的人。

如果你真的需要對方繼續解釋，請展示你已經做過的這一步。這樣對方知道你不是把下一輪思考也推回給他。

## 糟糕追問和好追問 [#糟糕追問和好追問]

比方說，如果我回答你：`看來似乎是 zentry 卡住了；你應該先清除它。`

這是一個**很糟的**後續回應：

`zentry 是什麼？`

更好的問法是：

`哦，我看過說明了，但是隻有 -z 和 -p 兩個參數中提到了 zentries，而且還都沒有清楚解釋如何清除它。你是指這兩個中的哪一個嗎？還是我看漏了什麼？`

## 追問也要繼續提供信息 [#追問也要繼續提供信息]

好的追問會說明三件事：你讀了什麼、你理解到了哪裡、你具體卡在哪個點。這樣別人只需要糾正你的理解或補充缺失部分，而不必重新從零開始教你。
