# 如何理解答案 (/zh-tw/docs/how-to-read-answer)



得到回答之後，真正的工作還沒有結束。你需要讀懂回答者給出的線索，判斷哪些內容是直接答案，哪些內容是在要求你繼續閱讀、搜索、補充信息或修正自己的假設。

## 回答不一定會像教程 [#回答不一定會像教程]

技術社區裡的回答常常很短，也可能很直接。它們不一定會把背景知識重新講一遍，而是指向手冊、關鍵詞、錯誤位置或下一步排查方向。你要學會從這些提示中繼續推進，而不是期待別人把完整學習路徑鋪好。

## 閱讀答案時先做三件事 [#閱讀答案時先做三件事]

1. 找出回答者指出的具體線索，例如文檔章節、命令、錯誤原因或關鍵詞。
2. 自己先閱讀、搜索和驗證，不要立刻要求對方展開講解。
3. 如果仍然不懂，再帶著你新學到的內容繼續追問。
