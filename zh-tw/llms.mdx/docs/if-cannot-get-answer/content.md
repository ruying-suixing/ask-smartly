# 如果無法得到解答 (/zh-tw/docs/if-cannot-get-answer)



如果仍得不到回答，請不要以為我們覺得無法幫助你。有時只是看到你問題的人不知道答案罷了。沒有回應不代表你被忽視，雖然不可否認這種差別很難區分。

## 不要立刻重複張貼 [#不要立刻重複張貼]

總的來說，簡單地重複張貼問題是個很糟的點子。這會被視為無意義的喧鬧。有點耐心，知道答案的人可能生活在不同的時區，可能正在睡覺，也可能暫時沒有看到你的問題。

也有可能，你的問題一開始就沒有組織好。與其原樣重發，不如先重新檢查標題、環境信息、復現步驟、錯誤輸出和你已經嘗試過的路徑。

## 換一個更合適的渠道 [#換一個更合適的渠道]

你可以通過其他渠道獲得幫助，而這些渠道通常更適合初學者。許多線上和本地用戶群組由熱情的軟件愛好者組成，他們未必親自寫過軟件，但很願意互相幫助，也更習慣回答入門問題。

如果問題屬於特定發行版、硬件、商業軟件、雲服務或組織內部環境，也應該優先尋找對應的用戶群、供應商支持、官方論壇或本地社區。換渠道不是逃避，而是把問題送到更可能有人理解上下文的地方。

## 接受付費支持也是正常選擇 [#接受付費支持也是正常選擇]

另外，你也可以向商業公司尋求幫助，不論公司大還是小。別因為需要付費才能獲得幫助而感到沮喪。假如你的汽車發動機汽缸密封圈爆掉了，你會把車送到修車鋪，併為維修付費。軟件沒有花錢，並不意味著技術支持也必須永久免費。

對於 Linux 這類大眾化軟件，每個開發者可能對應成千上萬名用戶。一個人不可能處理所有求助請求。即使你要為協助付費，和購買同類封閉軟件相比，成本通常也並不誇張；而且開源生態中的付費支持往往更直接、更透明。

## 重新整理後再回來 [#重新整理後再回來]

如果你仍想在原社區繼續提問，請先改進問題本身，而不是機械重複。把你新增的發現、縮小後的範圍、排除過的原因、仍然卡住的地方寫清楚。一個經過重新整理的問題，遠比同一段文字的第二次、第三次張貼更值得回應。
